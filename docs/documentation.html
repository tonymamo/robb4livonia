The /w and /q trigger alternate versions of the contribution form. You can also append ?iframe to trigger an alternate version of the contribution form that is designed to fit seamlessly into a website design. Examples of this are:

  • https://politics.raisethemoney.com/en/rdrzewicki?iframe=true

Standard Embed Code

This code allows you to embed the contribution for directly into your website. You will want to replace the iframe.src with a link to the contribution form you would like to use. Make sure and append ?iframe=true to the url to ensure a seamless design. 

  <script type="text/javascript">
    (function(){
      var scripts = document.getElementsByTagName('script'),
          thisScriptTag = scripts[scripts.length - 1],
          iframe = document.createElement('iframe');

      thisScriptTag.parentNode.insertBefore(iframe, thisScriptTag);
      iframe.src = 'https://politics.raisethemoney.com/en/rdrzewicki/w?iframe=true';
      iframe.width = '100%'; iframe.scrolling = 'no';
      iframe.setAttribute('seamless', true);
      iframe.setAttribute('frameBorder', 0);

      window.addEventListener('message', function(e) {
        iframe.height = e.data;
      });
    })();
  </script>

Embedding the Form in a WordPress website

The standard embed code does not work directly with a WordPress website, as they do not allow javascript snippets to be dropped directly into the content box. Our plugin enables you to drop a shortcode anywhere in your WordPress website to have the form displayed.

1. Download the WordPress Plugin
https://wordpress.org/plugins/raise-the-money/

2. Place the shortcode anywhere in your WordPress website:

Standard Contribution Form [raisethemoney site="politics" form="rdrzewicki"]
Wizard Contribution Form [raisethemoney site="politics" form="rdrzewicki" type="w"]
Quick Contribute Form [raisethemoney site="politics" form="rdrzewicki" type="q"]

The form attribute in the shortcodes above can be replaced with a campaign hash.

Custom form assets

Processing Javascript **REQUIRED**
The rtm-processing.js is required for processing contributions. This provides form validations, card tokenization, toggling ach and card payments, and contribution processing. 

<script src="rtm-processing.js"></script>

Theme Stylesheet **OPTIONAL**
The rtm-theme.css is not required. It provides basic styling of the contribution form very similar to what you would get with the form embed code. Without any styling the form is still 100% functional and will inherit the styles of your website. You can either create your own styles completely from scratch, or modify the RTM theme to closer match the design of your website. 

<link href="rtm-theme.css" media="all" rel="stylesheet">

Set values with hidden fields instead of text fields

Sometimes it is desirable to prefill the contribution amount instead of giving choices. This can be done by creating a hidden field with the amount filled in. So if you wanted to create a contribution form for people to contribute $25, you would remove all of the amount fields and instead add the following hidden input: 

<input type="hidden" name="contributor[amount]" value="25">

This technique can also be used for other fields, such as setting the frequency to one-time for all contributions or setting the duration to always go until next Election Day.

Adding custom fields to the contribution form

If you would like to add additional fields to the contribution, you can! Just set the input name to "custom[...]" with whatever you would like to call it inside of the square braces. Make sure and use lowercase with underscores separating the words for your input name: 

<input type="text" name="custom[your_name_here]">

Error messages

Field error messages will appear below the fields on validations. These will appear inside of <label class="error"> tags. Additionally, an .error class will be applied to invalid inputs for styling purposes.

Testing Contributions

For development purposes, you can set the form action to https://raisethemoney.com/testing. This will return validation errors or success, without actually processing transactions.
